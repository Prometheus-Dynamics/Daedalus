# Daedalus FFI Host

Host-side plugin installation and runner orchestration for the FFI design.

This crate owns the shared installer, schema-to-registry conversion, persistent worker process
runner, runner pool, state sync, and response decoding paths.

## Generated Fixture Harness

`run_scalar_add_generated_fixture_harness` runs the canonical scalar-add fixtures generated by
`ffi-core` through one Rust host path. The harness installs each generated schema/backend plan,
attaches a fake backend runner, invokes each fixture twice through `RunnerPool`, decodes
`InvokeResponse`, compares normalized outputs across languages, and records runner start/reuse
counts.

## Persistent Worker Runner

`PersistentWorkerRunner` is the host implementation of the JSON-lines worker protocol. It
starts a backend process once, expects a `WorkerHello` as the first stdout line, sends a
`WorkerProtocolAck`, then writes `WorkerMessage::Invoke` requests and reads event/response messages
without respawning the process. Stdout is reserved for protocol frames; stderr is captured and
capped for deterministic crash diagnostics.

The current regression coverage uses Python, Node, and Java worker fixtures that load their module
or classpath once and handle repeated invocations, state payloads, and event messages through the
same runner. Language SDKs can move to this runner incrementally without changing the `RunnerPool`
API.
